# AI 图文生成器 (AI Xiaohongshu Generator)

这是一个基于 React + Tailwind CSS 的图文生成工具，专为生成小红书/抖音风格的爆款图文设计。它本身是一个简单的 CMS，通过 AI 辅助快速生成内容页面。

## 🚀 核心玩法

这个项目的核心不是手写代码，而是**与 AI 协作**。

### 1. 准备工作
确保你拥有 `rule.md` 文件。这是项目的“宪法”，包含了所有的设计规范、安全区要求和代码结构定义。

### 2. 生成流程
当你想要制作一篇新的小红书图文时，只需要复制以下指令发送给 AI（如 ChatGPT、Claude、Trae AI 等）：

> "请阅读 `rule.md` 的规范。我需要制作一篇关于 [你的主题] 的图文。
> 文案内容/核心观点是：[你的文案或想法]
> 请帮我生成对应的 React 页面代码和文案配置。"

### 3. AI 会做什么？
AI 会根据 `rule.md` 的指引：
1.  自动在 `src/pages/` 下创建一个新的主题文件夹（如 `src/pages/my-new-topic/index.jsx`）。
2.  生成符合“垂直安全区”和“手机竖屏”视觉规范的 React 组件。
3.  自动撰写适配小红书风格的“标题党”文案和标签，并导出为 `meta` 对象。

### 4. 预览与发布
1.  保存 AI 生成的代码。
2.  回到网页（无需刷新），左侧侧边栏会自动出现新的页面标题。
3.  **点击列表**：在右侧预览图文效果。
4.  **点击“一键复制”**：获取左侧自动生成好的文案。
5.  **截图**：使用浏览器的“移动端模式”或直接截图右侧预览区域，保存为图片发布。

## 🛠️ 本地开发

```bash
# 安装依赖
npm install

# 启动项目
npm run dev
```

## 📂 项目结构

*   `src/pages/` - **内容核心区**。所有的图文页面都存放在这里，每个文件夹代表一篇笔记。
*   `rule.md` - **AI 规则书**。定义了视觉设计、代码模板和文案策略。
*   `src/App.jsx` - **CMS 主程序**。负责动态加载 `src/pages/` 下的所有内容并展示。

## ✨ 特性

*   **自动化路由**：基于文件系统的动态路由，新建文件即自动注册。
*   **安全区适配**：严格遵守小红书 UI 遮挡区域规范。
*   **文案管理**：左侧面板实时预览并一键复制文案。
*   **所见即所得**：1:1 还原手机端显示效果。
